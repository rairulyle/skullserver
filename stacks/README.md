# 💀 SKULLSERVER Docker Compose Stacks

Docker Compose stacks for SKULLSERVER, managed by Unraid Compose Manager.

## 📁 Directory Structure

```
stacks/
├── media/                       # 🎬 Media services
│   ├── docker-compose.yml
│   ├── .env                     # ⚙️ Auto-generated (do not edit)
│   ├── .env.example             # Stack variables template
│   └── .env.local               # 🔧 Stack variables (edit this)
├── infra/                       # 🌐 Network infrastructure
│   └── ...
├── ops/                         # ⚙️ Operations & comms
│   └── ...
└── o11y/                        # 📊 Observability
    └── ...
```

## 📦 Stack Overview

| Stack            | Description                              |
| ---------------- | ---------------------------------------- |
| 🎬 **media**     | Media server and content acquisition     |
| 🌐 **infra**     | Network infrastructure and reverse proxy |
| ⚙️ **ops**       | Operations and communications            |
| 📊 **o11y**      | Observability and monitoring             |

## ⚙️ Stack Management

### 🎛️ Unraid Compose Manager (Recommended)

All stacks are managed through the Unraid Compose Manager UI:

1. Navigate to **Docker** → **Compose Manager**
2. Select your stack
3. Use UI buttons to start/stop, update, view logs, or edit

`/boot/config/plugins/compose.manager/compose.manager.cfg`

```yaml
PROJECTS_FOLDER="/mnt/user/git/skullserver/stacks"
```

### 🔑 Naming Convention (Important)

For Compose Manager to recognize existing stacks, these **must** match:

- ✅ Folder name: `media`
- ✅ File name: `docker-compose.yml` (not `compose.yml`)
- ✅ Project name: `name: media` (at top of compose file)

Otherwise, duplicate folders like `media181998424` will be created.

### 💻 Docker Compose CLI (Alternative)

```bash
# Navigate to stack
cd /mnt/user/git/skullserver/stacks/media

# Start
docker compose up -d

# Update
docker compose pull && docker compose up -d

# Logs
docker compose logs -f [service_name]

# Stop
docker compose down
```

## 🔐 Environment Variables

### 📊 System Overview

```
Global Variables (stacks/.env)
    ↓
    ├─→ Stack Variables (stacks/media/.env.local)
    ├─→ Stack Variables (stacks/infra/.env.local)
    ├─→ Stack Variables (stacks/ops/.env.local)
    └─→ Stack Variables (stacks/o11y/.env.local)
           ↓
    Auto-Generated (stacks/[stack]/.env) ← Used by Docker Compose
```

### 🌍 Global Variables

**Location**: `stacks/.env`

Shared across all stacks. Retrieve from **Bitwarden → `SKULLSERVER Global Env`**

Common variables: `TZ`, `IP_ADDRESS`, `DOMAIN_NAME`, `APP_DATA`, `DATA`, etc.

### 🔧 Stack-Specific Variables

**Location**: `stacks/<stack>/.env.local`

Unique to each stack. Retrieve from **Bitwarden → `SKULLSERVER <stack> Env`**

Example for media stack: `PLEX_API_KEY`, `RADARR_API_KEY`, `SONARR_API_KEY`, etc.

### ⚙️ Auto-Generated Files

**Location**: `stacks/<stack>/.env`

**⚠️ DO NOT EDIT** - Auto-generated by `npm run sync-env`

Contains merged global + stack-specific variables used by Docker Compose.

## 🔄 Syncing Environment Variables

### Usage

```bash
npm run sync-env
```

### 📝 Workflow

1. **Edit Global** → `stacks/.env` (from Bitwarden)
2. **Edit Stack** → `stacks/<stack>/.env.local` (from Bitwarden)
3. **Sync** → `npm run sync-env`
4. **Deploy** → Restart stack in Compose Manager

### ➕ Adding New Variables

**Global Variables:**

1. Add to Bitwarden → `SKULLSERVER Global Env`
2. Add to `stacks/.env`
3. Add template to `stacks/.env.example`
4. Run `npm run sync-env`

**Stack-Specific Variables:**

1. Add to Bitwarden → `SKULLSERVER <stack> Env`
2. Add to `stacks/<stack>/.env.local`
3. Add template to `stacks/<stack>/.env.example`
4. Run `npm run sync-env`

## 🆕 Creating a New Stack

1. Create directory: `mkdir stacks/newstack`
2. Create `docker-compose.yml`:

   ```yaml
   name: newstack # Must match folder name

   services:
     myservice:
       container_name: myservice
       # ...
   ```

3. Create `.env.local` and `.env.example`
4. Run `npm run sync-env`
5. Import in Compose Manager UI

## 🐛 Troubleshooting

### ❌ Duplicate Folders Created

**Problem**: Compose Manager creates `media181998424` instead of using `media`

**Solution**: Ensure these match:

- ✅ Folder name: `media`
- ✅ File name: `docker-compose.yml` (not `compose.yml`)
- ✅ Project name: `name: media`

### ❌ Environment Variables Not Working

**Problem**: Services fail to start or can't find configuration

**Solution**:

1. Check `stacks/.env` exists
2. Check `stacks/<stack>/.env.local` has variables
3. Run `npm run sync-env`
4. Restart stack in Compose Manager

### ❌ Stack Not Detected

**Check**:

- File named `docker-compose.yml` (not `compose.yml`)
- File in `/mnt/user/git/skullserver/stacks/<stack>/`
- `name:` field at top of compose file

## 📋 Best Practices

- ✅ **Never edit** `<stack>/.env` (auto-generated)
- ✅ **Always run** `npm run sync-env` after changing env variables
- ✅ **Follow conventions** in [CONVENTIONS.md](./CONVENTIONS.md)
- ✅ **Use** `docker-compose.yml` (not `compose.yml`)
- ✅ **Set** `name:` field matching folder name
- ✅ **Keep secrets** in Bitwarden
